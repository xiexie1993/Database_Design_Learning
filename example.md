图标说明：
  1、 +-+
      | |  代表一个实体
      +-+
  2、 属性可是实体的惟一标识（也称主键），也可以是强制的或者任选的属性
      # 代表主键属性
      * 代表强制属
      o 代表任选属性
      若在同一实体框内则属于的共有属性
  3、
  4、 一个实体内的各个子类必须代表一个完整的分类集合（ 子类之和完全覆盖超类 )
  5、 

属性命名约定

属性名中的字符串                      含义
  ID                     标识           系统生成的连续的惟一的数字标识
  Seq_id                 顺序号         系统生成的父标识内的顺序号（例如，order_item_seq_id,订单条目顺序号）
  Code                   代码           用于标识用户定义的唯一标识的伪代码。其中可能包含着一些有意义的键（例如。存储Col_orado的地理代码可以是“CO”）
  Name                   名称           一个合适的代词，诸如人、地域、组织
  Description            描述           对于一个惟一代码或标识的描述性的值
  Flag 或 Ind(indicator) 标志           一个二元值(如，是/否，男/女)
  from_date              起始日期       用于描述包含于一个日期范围内的起始日期的属性
  thru_date              终止日期       用于描述一个日期范围内的结束日期的属性（不用to_date是因为thru_date能更清楚的描述日期范围的结束

~~~
